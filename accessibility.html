Accessibility Statement

Our Commitment to Accessibility

Sawsan Abdulbari is committed to ensuring digital accessibility for people with disabilities. I am continually improving the user experience for everyone and applying the relevant accessibility standards to ensure my portfolio website is accessible to all users.

This website has been designed and developed with accessibility as a core principle, following the Web Content Accessibility Guidelines (WCAG) 2.1 Level AA standards.

Conformance Status

The Web Content Accessibility Guidelines (WCAG) defines requirements for designers and developers to improve accessibility for people with disabilities. It defines three levels of conformance: Level A, Level AA, and Level AAA. This website is fully conformant with WCAG 2.1 level AA.

Accessibility Features

This website includes the following accessibility features to ensure all users can access and navigate the content effectively:

Navigation and Structure

  • Clear and consistent navigation structure across all pages
  • Skip to main content link on every page for keyboard users
  • Properly nested heading hierarchy (H1-H6) for easy navigation
  • Descriptive page titles that identify the page content and purpose
  • Breadcrumb navigation where appropriate

Keyboard Accessibility

  • All interactive elements are keyboard accessible
  • Visible focus indicators on all focusable elements
  • Logical tab order throughout the website
  • No keyboard traps - users can navigate in and out of all content
  • Custom keyboard shortcuts: Alt+H (Home), Alt+C (Contact), Alt+A (Accessibility)

Screen Reader Support

  • Semantic HTML5 elements for proper structure
  • ARIA landmarks for main page regions
  • ARIA labels and descriptions where needed
  • Form labels properly associated with form controls
  • Error messages announced to screen readers
  • Live regions for dynamic content updates

Visual Design

  • High contrast colors meeting WCAG AAA standards
  • Text can be resized up to 200% without loss of functionality
  • No reliance on color alone to convey information
  • Consistent and predictable layout
  • Support for dark mode based on user preferences

Images and Media

  • Alternative text for all informative images
  • Captions and transcripts for video content
  • Audio descriptions where appropriate
  • Images of text avoided where possible

Forms

  • Clear labels for all form inputs
  • Required fields clearly marked
  • Helpful error messages and validation
  • Fieldsets and legends to group related inputs
  • Instructions and help text where needed

Responsive Design

  • Fully responsive layout that works on all devices
  • Touch targets of at least 44x44 pixels
  • Content reflows for different screen sizes
  • No horizontal scrolling required

Cognitive Accessibility

  • Clear and simple language
  • Consistent navigation and layout
  • Important information highlighted appropriately
  • No time limits on content
  • Options to pause or stop moving content
  • Clear instructions for interactive elements
  • Predictable functionality - no unexpected changes

Additional Features

  • Respects user's motion preferences (reduced motion)
  • Print-friendly styles for all content
  • Progressive enhancement approach
  • Works without JavaScript enabled

Technologies and Standards

This website is built using:

  • Semantic HTML5 for structure
  • CSS3 with custom properties for theming
  • Vanilla JavaScript for enhanced functionality
  • ARIA (Accessible Rich Internet Applications) for complex interactions

The website follows these standards:

  • WCAG 2.1 Level AA compliance
  • Section 508 compliance
  • ARIA Authoring Practices Guide
  • HTML5 validation

Testing and Evaluation

This website has been thoroughly tested for accessibility using a combination of automated tools, manual testing, and assistive technology testing:

Automated Testing Tools

  • axe DevTools for comprehensive accessibility auditing
  • WAVE (WebAIM's Web Accessibility Evaluation Tool)
  • Lighthouse (Google Chrome DevTools)
  • Pa11y command line tool for automated testing

Manual Testing

  • Keyboard-only navigation testing
  • Screen reader testing with NVDA (Windows) and VoiceOver (macOS)
  • Color contrast verification
  • Text resize and zoom testing (up to 200%)
  • Mobile device testing on various screen sizes

Browser Compatibility

The website has been tested and works in:

  • Chrome (latest version)
  • Firefox (latest version)
  • Safari (latest version)
  • Edge (latest version)

Known Limitations

While we strive for complete accessibility, some areas may have limitations:

  • Some older PDF documents linked from the site may not be fully accessible. We are working to update these or provide accessible alternatives.
  • Third-party embedded content (such as social media widgets) may not meet the same accessibility standards as the rest of the site.
